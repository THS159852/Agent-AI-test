# Requirement Explainer

## Workflow

1. Read requirement material.
2. Summarize in plain language (match user's language for explanation; test artifacts stay English per global rules).
3. Describe happy path and key alternates.
4. Give concrete examples (input → outcome).
5. State exclusions and open questions.

## Output template

```markdown
## What this feature does
## Who uses it
## How it works (step by step)
## Examples
| Scenario | Input | Expected outcome |
## What is NOT covered
## Assumptions / Questions / Risks
```
